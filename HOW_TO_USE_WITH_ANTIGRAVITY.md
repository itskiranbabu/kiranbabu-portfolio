## üí° BEST PRACTICES FOR WORKING WITH ANTIGRAVITY

### 1. Be Specific

‚ùå Bad: "Add SEO to my site"
‚úÖ Good: "Add Open Graph tags, Twitter Cards, and Schema.org Person markup for SEO"

### 2. Provide Context

Always include:
- Repository name
- Current file structure
- Relevant code sections
- Specific requirements

### 3. Ask for Complete Code

Request:
- Full HTML, CSS, JavaScript
- File names and locations
- Testing instructions
- Assets needed

### 4. Implement Incrementally

- Start with one feature
- Test thoroughly
- Commit to Git
- Move to next feature

### 5. Test Everything

After each implementation:
- Test on mobile
- Test in different browsers
- Run Lighthouse audit
- Check accessibility

### 6. Ask for Clarification

If something is unclear:
```
Can you explain how the GitHub API rate limiting works in this implementation?
```

### 7. Request Modifications

If you need changes:
```
Can you modify the timeline to show only 2 experiences instead of 3?
```
